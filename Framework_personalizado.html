Framework personalizado de segurança e governança de TI

Este Framework tem como objetivo fornecer uma base para utilizarmos as boas práticas dos frameworks COBIT, ISO/IEC 27001 e NIST.

1. Governança Estratégica
Base: COBIT 2019 + NIST CSF 2.0 (Govern)

A instalação do MySQL deve ser realizada de acordo com as necessidades do ambiente de produção, garantindo alta disponibilidade, desempenho e segurança. Abaixo estão os passos principais:

  • Alinhamento aos stakeholders: Definir metas de segurança alinhadas aos objetivos de negócio, utilizando o princípio "Meeting stakeholder needs" do COBIT.
  • Estrutura de governança: Combinar as categorias do NIST CSF 2.0 (GV.RM para gestão de riscos, GV.RR para papéis) com os enablers do COBIT (políticas, estruturas organizacionais)
  • Ferramenta prática: 
    Comitê de Governança:
  - Diretoria: Define políticas e orçamentos (COBIT APO01)
  - Equipe de Segurança: Implementa controles (ISO 27001 A.5) e monitora ameaças (NIST Detect)

2. Gestão de Riscos e Controles
Base: ISO 27001 + NIST Protect/Detect

Todo Framework necessita que sejam mensurados os riscos e quais ações serão tomadas caso aconteça algum incidente.

  • Avaliação de riscos: Implementar a cláusula 6.1 da ISO 27001 para identificar ameaças e vulnerabilidades, integrada ao Identify do NIST.
  • Controles prioritários: 
      Criptografia de dados (ISO A.8.2, NIST PR.DS)
      Gestão de acesso (ISO A.9.1, NIST PR.AC)
      Backup e recuperação (ISO A.12.3, NIST RS.RP)
  • Mapeamento cruzado:
        ISO 27001:2022         NIST CSF 2.0    COBIT 2019
    A.5.1 (Políticas) GV.PO (Govern)   APO01 (Governança)   
    A.12.4 (Logs)     DE.AE (Detect)      DSS02 (Monitoração)   

3. Operações de Segurança
Base: NIST Respond/Recover + ISO 27001

  • Resposta a incidentes: 
      Plano alinhado ao NIST Respond (notificação, contenção)
      Integração com a cláusula 8.2 da ISO (tratamento de não conformidades)
  • Resiliência: 
      Testes de recuperação de desastres (NIST RC.RP)
      Análise pós-incidente (ISO 27001 A.16.1)

4. Conformidade e Melhoria Contínua
Base: COBIT + ISO 27001

  • Monitoramento: 
      Métricas de maturidade do COBIT (ex: nível 3 em APO13)
      Auditorias internas da ISO 27001 (cláusula 9.2)
  • Certificação: Alinhar o SGSI da ISO 27001 com os objetivos de governança do COBIT

5. Implementação Prática

  • Diagnóstico inicial: 
      Mapear processos existentes com COBIT
      Realizar avaliação de riscos conforme ISO 27001
  • Priorização: 
      Usar o Current State vs. Target State do NIST para gaps críticos
  • Piloto: 
      Implementar em um departamento (ex: TI) controles como:
        Gestão de acesso (ISO A.9.1 + NIST PR.AC)
        Backup automatizado (NIST PR.IP + ISO A.12.3)

Vantagens da Integração

  • Custos reduzidos: Elimina redundâncias entre frameworks
  • Resiliência ampliada: Combina governança (COBIT), controles (ISO) e resposta (NIST)
  • Conformidade regulatória: Atende LGPD, GDPR e leis setoriais

  • Para adaptação à realidade organizacional, recomenda-se iniciar com um roadmap de 6-12 meses, priorizando áreas de maior exposição a riscos
